*Улучшенный диспетчер задач* - это Python-приложение с графическим интерфейсом (GUI) для мониторинга и управления системными процессами. Приложение предоставляет:

·        Список всех запущенных процессов с детальной информацией

·        Графики использования CPU и памяти

·        Возможность завершения процессов

·        Фильтрацию и сортировку процессов

·        Детальную информацию о каждом процессе

2. Технические требования

·        Python 3.7+

·        Зависимости:

o   tkinter (обычно входит в стандартную поставку Python)

o   psutil (для работы с процессами)

o   matplotlib (для построения графиков)

3. Архитектура

3.1 Основные компоненты

1.     **ProcessCache** - реализует кэширование информации о процессах для оптимизации

2.     **ProcessManager** - основной класс для работы с процессами

3.     **TaskManagerApp** - главный класс приложения, реализует GUI

4. Детальное описание компонентов

4.1 ProcessCache

**Назначение**: Оптимизирует работу с процессами за счет кэширования.

**Основные методы**:

·        update(processes) - обновляет кэш

·        needs_full_update(current_pids) - определяет, нужно ли полное обновление

**Поля**:

·        _cache: OrderedDict для хранения кэшированных процессов

·        max_size: максимальный размер кэша (по умолчанию 500)

·        _static_fields: статические поля процесса (не изменяются)

·        _dynamic_fields: динамические поля (изменяются часто)

4.2 ProcessManager

**Назначение**: Управляет процессами и предоставляет API для работы с ними.

**Основные методы**:

·        get_processes(full_update) - возвращает список процессов

·        _refresh_dynamic_data() - обновляет динамические данные

·        kill_process(pid) - завершает процесс по PID

4.3 TaskManagerApp

**Назначение**: Реализует графический интерфейс и основную логику приложения.

**Основные методы**:

·        setup_ui() - инициализирует интерфейс

·        refresh_processes() - обновляет список процессов

·        update_charts() - обновляет графики CPU и памяти

·        show_process_details() - показывает детали процесса

·        kill_selected_process() - завершает выбранный процесс

5.1 Основное окно

2.     Графики:

ПКМ на процессе открывает меню с опциями:

·        Подробности - показывает детальную информацию

6. Алгоритмы и оптимизации

6.1 Кэширование процессов

·        Статические данные (имя, PID, пользователь) кэшируются

·        Динамические данные (CPU, память) обновляются чаще

·        Полное обновление происходит только при:

o   Первом запуске

o   Завершении процесса

o   Изменении более 5 процессов

o   Прошествии 5 секунд с последнего полного обновления

6.2 Обновление интерфейса

·        Графики обновляются каждую секунду

·        Список процессов обновляется каждые 2 секунды

·        Системная информация обновляется каждую секунду

7. Обработка ошибок

Приложение обрабатывает:

·        Отсутствие необходимых модулей

·        Ошибки доступа к процессам

·        Ошибки при завершении процессов

·        Ошибки обновления интерфейса

В DEBUG-режиме (DEBUG = True) выводятся подробные сообщения об ошибках.

# **ProcessInfo.dll - Техническая документация**

## Основное назначение

DLL для Windows, предоставляющая информацию о процессах в формате JSON.

## Как собираются данные

1.     Получает список процессов через `EnumProcesses`

2.     Для каждого процесса:

o   Получает имя через `GetModuleBaseNameA`

o   Измеряет используемую память через `GetProcessMemoryInfo`

o   Пропускает системные процессы (PID=0) и недоступные процессы

## Управление памятью

·        Вызывающая сторона ОБЯЗАНА освобождать память через `FreeProcessInfoMemory`

·        Использует WinAPI `LocalAlloc`/`LocalFree`

## Обработка ошибок

·        Возвращает пустой массив `[]`, если не удалось получить список процессов

·        Возвращает `nullptr` при ошибках выделения памяти

·        Все ошибки WinAPI подавляются

## Производительность

·        Минимальные накладные расходы - чистый WinAPI без лишних прослоек
 **Мониторинг процессов**

- Программа собирает информацию о всех запущенных процессах:

- **PID** — идентификатор процесса.
- **Имя процесса** — название исполняемого файла.
- **Использование CPU** — процент использования процессора.
- **Использование памяти** — объем оперативной памяти, используемой процессом (в МБ).
- **Путь к исполняемому файлу** — полный путь к файлу процесса.

**2. Завершение процессов**

- Пользователь может ввести PID процесса, чтобы завершить его. Программа использует функцию TerminateProcess для принудительного завершения.

**3. Многопоточность**

- Программа обновляет информацию о процессах в фоновом режиме каждые 2 секунды с помощью отдельного потока.

**Структура программы**

**1. Основные файлы**

- **main.cpp**:

- Содержит точку входа в программу (main).
- Реализует консольный интерфейс для взаимодействия с пользователем.
- Запускает фоновый поток для обновления информации о процессах.

- **process_info.h**:

- Заголовочный файл, содержащий объявления классов и структур:

- ProcessInfo — структура для хранения информации о процессе.
- ProcessManager — класс для управления процессами.

- **process_info.cpp**:

- Реализация методов класса ProcessManager:

- Сбор информации о процессах.
- Расчет использования CPU и памяти.
- Завершение процессов.

**2. Зависимости**

- Программа использует WinAPI для работы с процессами и многопоточности:
- <windows.h> — для вызова функций WinAPI.
- <tlhelp32.h> — для создания снимков процессов.
- <psapi.h> — для получения информации о памяти.
- <thread> — для многопоточности.
- <iostream> — для ввода/вывода в консоль.



---

**Как работает программа**

1. **Инициализация**:

- При запуске программа повышает привилегии для доступа к процессам с помощью функции EnableDebugPrivilege.
- Создается объект ProcessManager, который начинает сбор информации о процессах.

3. **Фоновое обновление**:

- В отдельном потоке каждые 2 секунды вызывается метод ProcessManager::update(), который обновляет информацию о процессах.

5. **Консольный интерфейс**:

- Пользователь может выбрать одну из опций:

- **1. Показать процессы** — выводит список процессов.
- **2. Завершить процесс** — завершает процесс по введенному PID.
- **3. Выход** — завершает работу программы.

**Пример использования**

1. Запустите программу.
2. В консоли появится меню:

Copy

1. Показать процессы

2. Завершить процесс

3. Выход

4. Выберите опцию:

- Введите 1, чтобы увидеть список процессов.

**Технические детали**

**1. Класс** **ProcessManager**

- **Методы**:

- update() — обновляет информацию о процессах.
- kill_process(int pid) — завершает процесс по PID.
- get_processes() — возвращает список процессов.

- **Поля**:

- processes — словарь с информацией о процессах.
- mtx — мьютекс для синхронизации доступа к данным.

**2. Структура** **ProcessInfo**

- Содержит информацию о процессе:

- pid — идентификатор процесса.
- name — имя процесса.
- memory_usage — использование памяти (в байтах).
- cpu_usage — использование CPU (в процентах).
- exec_path — путь к исполняемому файлу.

**3. Многопоточность**

- Для обновления информации о процессах используется отдельный поток:
```
std::thread([&pm]() {

    while (true) {

        pm.update();

        std::this_thread::sleep_for(std::chrono::seconds(2));

    }

}).detach();

```
